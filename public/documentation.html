This API builds a server-side component of the myFlix API
Request URL HTTP METHOD RESPONSE
Return a list of ALL movies to the user /movies GET

A JSON object holding data about all the movies
Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user /movies/[title] GET

A JSON object holding data about a single movie, containing a description, genre, director, image url, and whether it's featured or not.

Return data about a genre (description) by name/title (e.g., “Thriller”) /movies/[genre] GET

A JSON object holding data about a genre, containing a description.

Return data about a director (bio, birth year, death year) by name /movies/[director_name] GET

A JSON object holding data about a director, containing a bio, birth year, and death year.

Allow new users to register /users POST

A JSON object holding data about the new user that was added including a username.

Allow users to update their user info (username) /users/[username]/about PUT

A text message indicating the user info was updated
Allow users to add a movie to their list of favorites (showing only a text that a movie has been added—more on this later) /users/[username]/profile PUT

A JSON object holding data about the movie that was added to favorites.

Allow users to remove a movie from their list of favorites (showing only a text that a movie has been removed—more on this later) /users/[username]/profile DELETE

A text message indicating whether the movie was successfully removed from their favorites.

Allow existing users to deregister (showing only a text that a user email has been removed—more on this later) /users/[username] DELETE

A text message indicating whether or not the username was deregistered.
